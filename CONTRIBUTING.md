Thank you for considering contributing to Voidr's technical assessments! 

## How to Contribute

### Reporting Bugs or Issues

If you found a problem with the tests:

1. Check if the issue hasn't already been reported in [Issues](../../issues)
2. Open a new issue with:
   - Clear and descriptive title
   - Detailed description of the problem
   - Steps to reproduce (if applicable)
   - Expected behavior vs actual behavior

### Suggesting Improvements

Have ideas to improve the tests?

1. Open an issue describing:
   - Your suggestion in detail
   - Why this improvement would be useful
   - Implementation examples (if possible)

### Submitting Pull Requests

1. Fork the repository
2. Clone your fork locally
3. Create a branch for your contribution:
   ```bash
   git checkout -b feature/improvement-name
   ```
4. Make your changes
5. Commit your changes with clear messages:
   ```bash
   git commit -m "Add: clear description of the change"
   ```
6. Push to your fork:
   ```bash
   git push origin feature/improvement-name
   ```
7. Open a Pull Request to the main repository

### Code Standards

- Keep code clean and well documented
- Use descriptive names for variables and functions
- Add comments when necessary
- Follow language-specific conventions

### Adding New Tests

If you're a Voidr team member adding a new test:

1. Create a new folder in the appropriate category (frontend, backend, sdet, devops)
2. Include a detailed README.md with:
   - Challenge description
   - Learning objectives
   - Technical requirements
   - Evaluation criteria
   - Submission instructions
   - Estimated time
3. Add examples or templates when appropriate
4. Update the main README

## Code of Conduct

- Be respectful and constructive
- Accept constructive criticism
- Focus on what's best for the community
- Keep discussions productive and professional

See our full [Code of Conduct](./CODE_OF_CONDUCT.md) for more details.

## Questions?

If you have questions about contributing, open an issue with the `question` tag.
